Content-Grid System

ACSS Content-Grid provides named grid lines for sophisticated layout control:

Grid Template Structure

[full-start] gutter [feature-max-start] [feature-start] [content-start] content [content-end] [feature-end] [feature-max-end] gutter [full-end]
Content Grid Classes:
.content-grid - Enable content-grid on container
.content--feature - Span feature columns (wider than content)
.content--feature-max - Span feature-max columns (widest)
.content--full - Span full width (edge to edge)
.content--full-safe - Span full width with gutter padding
Mathematical Precision

Content-grid uses named grid lines and sophisticated minmax() calculations to provide precise control over content width and breakout behavior. Content automatically centers while breakout elements can expand to feature or full-width areas.

Default Content

This content respects the content-grid's default content width. It's constrained to a readable line length and centered within the container. Perfect for articles, blog posts, and standard text content.

No class needed - default behavior

Feature Content

This content extends into the feature areas, making it wider than regular content. Great for highlighted sections, pull quotes, or special callout boxes that need more visual prominence.

.content--feature

Feature-Max Content

This extends even further into the feature-max areas. Ideal for images, videos, code blocks, or any content that benefits from more horizontal space while still maintaining some edge padding.

.content--feature-max

Full-Width Content

This content spans the entire width of the container, edge to edge. Perfect for hero sections, full-width images, or any content that needs maximum visual impact. Notice how it breaks out of the grid completely.

.content--full

Full-Safe Content

This content is full-width but maintains safe padding on the edges. It respects the gutter areas, ensuring content doesn't touch the viewport edges on mobile devices. Best for sections that need to feel full-width but remain readable.

.content--full-safe

Default Content Block

This entire block is a single child of content-grid, so it uses the default content width. Notice how it's narrower than the feature and feature-max areas above.

Regular paragraph text continues at the default content width, maintaining optimal readability.

Full-width callout section

Pro tip: The content-grid system automatically handles responsive behavior. On smaller screens, the feature and feature-max areas shrink proportionally, ensuring content remains readable and well-proportioned across all devices.
